### **5-1. 바이브코딩이란?**

<aside>
💡

**Vibe Coding = "Vibe"(느낌, 분위기) + "Coding"(코딩)**

코드를 직접 작성하지 않고, **자연어로 대화하며 프로그램을 만드는 방식**입니다.

</aside>

### **5-2. 전통적 개발 vs AI 바이브코딩**

| 비교 항목 | 전통적 개발 | AI 바이브코딩 | 개선율 |
| --- | --- | --- | --- |
| **소요 시간** | 3-4주 | 1-2일 | ⚡ 90% 단축 |
| **개발 비용** | 300~500만원 | 월 $20 (AI 도구) | 💰 99% 절감 |
| **필요 기술** | 프로그래밍 언어 숙련 | 한국어로 설명만 가능하면 OK | 📚 진입장벽 제로 |
| **수정 용이성** | 개발자 재투입 필요 | 대화로 즉시 수정 | 🔧 즉각 대응 |

**👨‍🏫 왜 중요한가요?**

이제 여러분도 **데이터 분석 도구, 자동화 스크립트, 간단한 웹사이트**를 직접 만들 수 있습니다. 개발자를 기다리지 않아도 됩니다. 이것이 **AI 시대의 업무 혁신**입니다.

## **6. 🚨 AI의 한계와 안전한 사용법**

AI는 강력한 도구지만 만능은 아닙니다. 반드시 알아야 할 한계와 주의사항을 짚고 넘어가겠습니다.

### **6-1. ⚠️ 환각(Hallucination) 현상**

<aside>
🚨

**환각(Hallucination)이란?**

AI가 사실이 아닌 정보를 마치 사실인 것처럼 그럴듯하게 생성하는 현상입니다.

</aside>

**실제 사례:**

- ❌ 존재하지 않는 통계 데이터 생성
    - 예: "2024년 바이오헬스 시장은 350조원"
    - 실제: 데이터 없음, AI가 지어낸 수치
- ❌ 가짜 논문 인용
    - 예: "Kim et al.(2023)의 연구에 따르면..."
    - 실제: 해당 논문 존재하지 않음
- ❌ 없는 기업·제품 생성
    - 예: "ABC바이오텍의 XYZ-500 제품이 FDA 승인"
    - 실제: 그런 회사/제품 없음

**발생 빈도:**

- 최상위 모델 (GPT-5, Claude 3.5): 약 1.4~2.5% 발생
- 일반 LLM: 약 3~5% 발생
- **특히 통계, 인용, 전문 분야에서 빈번**

**3단계 검증 방법:**

```
1단계: 출처 요구
→ "각 데이터의 출처와 발행 연도를 명시하세요"
→ "인용한 논문의 저자, 제목, 저널명을 알려주세요"

2단계: 교차 확인
→ ChatGPT + Claude + Perplexity로 동일 질문
→ 3개 모두 같은 답 → 신뢰도 ↑
→ 답이 다름 → 의심 필요

3단계: 공식 출처 대조
→ 통계청, 한국은행, 산업연구원 등 공식 사이트 확인
→ 논문: Google Scholar, DBpia 검색
→ 기업 정보: 공시자료, 공식 홈페이지
```
